## Overview

This document describes the survival system integration work completed in this cycle.
The `SurvivalComponent` is the runtime subsystem that tracks player survival stats
(health, hunger, thirst, stamina, fear) and feeds them into gameplay consequences.

Since the UE5 Editor binary is pre-built (no live C++ recompilation), all integration
work is done via Python/Blueprint-level wiring and actor tagging in the MinPlayableMap.

## Survival Threat Proximity Rules (Design Spec)

These rules should be implemented in Blueprint (BP_TranspersonalCharacter):

| Condition | Effect |
|-----------|--------|
| Within 800u of any `Predator` actor | `FearLevel` += 5/sec |
| Within 400u of `TRex_Savana_001` | `FearLevel` += 15/sec, `Stamina` drain x2 |
| Within 200u of any `Predator` | `Health` -= 10/sec (attack range) |
| No predator within 1200u | `FearLevel` -= 2/sec (recovery) |
| `Hunger` < 20 | `Stamina` max cap = 50 |
| `Thirst` < 10 | `Health` -= 1/sec |
