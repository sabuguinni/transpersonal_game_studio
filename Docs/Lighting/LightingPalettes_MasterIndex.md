## CAP Enforcement Rules (Applied Every Cycle)

All palettes enforce these constraints before applying changes:

1. **Sun Pitch Guard** — Never above -30° (prevents sun going underground)
2. **Fog Dedup** — Maximum 1 ExponentialHeightFog actor in scene
3. **FastSkyLUT** — `r.SkyAtmosphere.FastSkyLUT 1` always enabled
4. **SkyLight** — `real_time_capture = True` always active
5. **Map Save** — `save_current_level()` always called after changes

## Lighting Design Philosophy

> "The player doesn't notice good lighting. They only notice bad lighting."
> — Roger Deakins principle applied to game cinematography

Each palette serves a **survival gameplay function**:

- **Dawn** — New day, new dangers. Hope mixed with threat.
- **Midday** — Maximum visibility, maximum heat exhaustion risk.
- **Dusk** — Time pressure. Night predators awakening. Beautiful but deadly.
- **Storm** — Zero visibility. Disorientation. Survival emergency.
- **Night** — Stealth or shelter. Bioluminescence as navigation.

## Technical Architecture

### Light Actors Spawned Per Palette

```
Dawn:   4 actors (2 SpotLights + 2 PointLights)
Midday: 3 actors (SkyLight bounce + 2 PointLights)
Storm:  2 actors (Lightning PointLight + Rain SpotLight)
Dusk:   3 actors (WestFill PointLight + EastShadow PointLight + GroundBounce RectLight)
```

### Naming Convention
All lighting actors follow: `Light_[Palette]_[Type]_[NNN]`
- `Light_Dusk_WestFill_001`
- `Light_Dawn_RuinShaft_001`
- `Light_Storm_Lightning_001`
